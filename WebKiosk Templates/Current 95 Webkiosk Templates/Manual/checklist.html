Setup Checklist

Use this checklist to make sure you have followed all of the steps needed to create your kiosk.

  • Use EmbARK to describe your collection thoroughly
    • Catalog artist and object records
    • Apply keywords to records (optional)
    • Create links among object, artist, and surrogate records
  • Identify the records you want to include in the kiosk.
  • Mark these records for web access.
  • Designate long text Notes fields for web access (optional)
  • Make sure the images attached to the records are compatible with the Kiosk:
    • The files are in a web-compatible format (JPG, GIF, or PNG)
    • The file names can be read by web browsers (for example, My_image.jpg rather than My Image with no file extension; use EmbARK Rename function to batch change file names.)
    • All images and image folders are contained in a master folder called Images (or in another folder that you must identify in the kiosk preferences)
  • Group object records into portfolios and mark the portfolios for web access.
  • Design an Index Tree and link portfolios to it. (optional)
  • Choose object and artist Quick Search fields:
    • Run Unique Values freport on fields to check for inconsistent data entry
    • Use Replace to fix any inconsistent entries
    • Update Quick Search indices
  • Choose Contains search fields and update Contains Search indices (for the Google-like search in the side bar).
  • Copy the EmbARK datafile and Images folder to the server computer (or configure Prefs for an alternate image server).
  • Consider using Zoomify to enable zooming of high-resolution files.
  • Open the Kiosk application on the web server computer.
  • Set the kiosk preferences and customize the page templates (as desired). Be sure to take advantage of Development Mode.
  • Adjust the header.inc include file to insert a custom logo and adjust the left menu items.
  • Adjust the kioskstyles.css cascading style sheet in the webfolder to quickly change overall site colors and font properties.
  • Refresh the datafile as necessary by copying from the primary EmbARK site to the web server and restarting.
