Introduction

The EmbARK Web Kiosk lets you make all or part of your EmbARK collection available for viewing on a web browser, either over the Internet or through your institution’s local network.

Visitors to your kiosk will be able to view objects and data from your collection by selecting "paths" or portfolios that you have set up, or by searching for records based on keywords or specific artist/object information.

In each case, the kiosk merges the chosen records and images with predesigned web page templates to present the results in an attractive format. You can customize these templates to match the appearance of your institution’s website.

This supplement is presented in seven sections:
  • Setup Checklist provides a suggested sequence of steps for a Web Kiosk implemenation.
  • Preparing the Database explains how to use EmbARK to identify the records and images to be included in the EmbARK Web Kiosk, and contains cataloging tips that will make the kiosk more effective.
  • Setting Up the Server describes how to install the EmbARK Web Kiosk software on your web server.
  • Maintaining the Kiosk details tasks that should be performed occasionally to keep your EmbARK Web Kiosk running and updated properly.
  • Customizing the Kiosk reviews the HTML templates and commands used in the EmbARK Web Kiosk, and outlines how to customize the templates to work with your datafile and your overall website.
  • Kiosk Tags is a listing of each of the codes that can be used for displaying information on the kiosk pages.
  • 4D Tags is a listing of advanced scripting tools that can be used in addition to Kiosk Tags.
  • XML Data Sharing provides details regarding tools for generating custom data output (such as XML or SHTML) and using advanced URL queries to return that data set. This is an ADVANCED feature.
  • Appendix provides information on additional topics such as image zooming with Zoomify and running the kiosk as an automatic service.


Note: This manual makes reference to default viewed table and field names, such as Object, Artist, or Display Title. It is quite likely that table or field names may have been customized in your own database and not match references in this guide.
